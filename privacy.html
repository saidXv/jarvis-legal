Jarvis Discord Bot

Privacy Policy

Effective and last updated: July 4, 2026

This policy explains what data Jarvis processes, why it is used, how it is shared and retained, and how users can request access or deletion.

1. Data we collect and store

  • Discord identifiers and profile data made available to the bot, including user ID, username, discriminator where present, avatar reference, and activity timestamps.
  • Game data, including cards, teams, currencies, inventory, weapons, statistics, cooldowns, quests, rankings, achievements and related progression.
  • Vote history, vote source, streaks, rewards and opt-in notification settings.
  • Notes, reminder text, titles and tags that a user deliberately asks Jarvis to save.
  • Relevant server and channel identifiers and settings. Server records may include server name and ID, owner ID and username, member count, invite, and join or leave status.

2. Message Content privileged intent

Jarvis processes message content for these user-facing functions:

  • Recognizing j and jarvis commands and parsing their arguments.
  • Processing eligible message activity in the configured community server for rank XP and daily message quests.
  • Scanning messages only in the configured moderation channel for configured prohibited terms.

When automod acts, it deletes the triggering message and sends up to 1,000 characters of that message, the matched term, and relevant user, channel and message identifiers to a restricted moderation log channel on Discord. Ordinary chat history is not stored in MongoDB. Content deliberately submitted to the Notes or Reminders features is stored so those features can operate.

3. How we use data

  • Execute commands and maintain user game progress.
  • Operate teams, battles, trading, leaderboards, quests, rewards, notes and reminders.
  • Deliver vote reminders and other notifications that users enable.
  • Moderate the configured channel and investigate abuse.
  • Maintain server configuration, security, cooldowns and anti-exploit controls.

4. Service providers and sharing

Jarvis uses Discord to receive and send data, Cybrancee for application hosting, and MongoDB for database storage. Top.gg and DiscordBotList provide vote events. These providers process data only as needed to perform their role. Data may also be disclosed when required by law or to investigate security and abuse.

We do not sell personal data or share it with advertisers or data brokers.

5. Retention

Data is retained while needed to operate Jarvis or until a valid deletion request is completed. Game progress, completed reminders, vote history and server history currently have no fixed automatic deletion period. Automod logs remain in Discord until removed by server moderators. Data that is no longer necessary will be deleted, and data will also be deleted when required by Discord or applicable law.

6. Security

Database access is authenticated and restricted, administrative access is limited, and credentials are stored outside the public source repository. No online service can guarantee absolute security.

7. Your choices and requests

Users can use Jarvis commands to view or remove their notes and reminders and to manage vote notifications. To request access, correction, export or deletion of stored data, contact the bot administrators through the Jarvis support server. We aim to respond to valid requests within 30 days.

Stopping command use does not erase existing records. Messages in configured XP, quest or automod locations may still be processed. Some security or server records may be retained when legitimately required; any exception will be explained.

8. Data we do not request

Jarvis does not request Discord passwords, account tokens, email addresses, phone numbers, payment details, precise location, device identifiers, voice, or video. Never submit credentials through commands, notes or reminders.

9. Children

Users must meet Discord’s minimum age requirement for their country. Jarvis is not directed to anyone below that minimum age.

10. Changes and contact

This policy may be updated when Jarvis features or legal requirements change. The date at the top identifies the latest version. Questions and privacy requests can be submitted through the support server.
